Inline 3D in the page

CEF offscreen-render → the host drives the real display-processor weave through XR_DXR_weave. The host never weaves; each 3D element below is a side-by-side canvas whose flat pixels are replaced by the woven 3D in the host's present surface. The page has multiple inline-3D elements at once — mixed 2D/3D is driven straight from the DOM (one .element3d canvas per region; the host weaves each per frame). Scroll, zoom, and drag the window — every lattice must stay locked.

First 3D element

This is ordinary 2D DOM content forming the surround. It scrolls normally. Each 3D element participates in page flow, so scrolling moves it — the host tracks its committed device-pixel rect every frame and feeds it to the weave RPC as the window-relative rect.

As the viewer's head moves, the runtime's tracked eyes flow back to this page, which re-renders every element off-axis (look-around) using an asymmetric-frustum projection.

Second 3D element

A second independent inline-3D region with its own scene (a single cube that pops well in front of the screen plane). It is weaved by a separate per-element weave call into its own committed rect, composited over the same present surface as the first.

Third 3D element

A third region, deeper behind the screen. Add or remove a .element3d canvas and the set of 3D regions changes with no host-side bookkeeping — the DOM is the zone description.

Below the elements

More surround text so the page is comfortably scrollable. Use the mouse wheel or Page Down to scroll the 3D elements across the viewport while watching the live display for any lattice drift or 3D collapse.
